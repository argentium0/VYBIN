Mobile Application Development Course Project
General Project Description
The objective of this course project is to design and develop a complete cross-platform mobile application using Flutter and Firebase. The project will be completed in four phases, allowing students to gradually learn frontend development, application navigation, backend integration, and device service implementation. Each phase builds upon the previous one to provide practical experience in modern mobile application development.

Objective
Develop a fully functional user interface using Flutter by implementing fundamental widgets and layout concepts.

Requirements

Responsive Layout using MediaQuery and Expanded/Flexible

Complete frontend UI

Responsive design

Phase 2: Navigation and Application Flow
Objective
Connect all application screens through Flutter navigation.

Requirements

Returning Data from Screens

Students should create a complete application flow where every screen is interconnected.

Project Evaluation Criteria
Component	Weightage
Phase 1: Flutter UI Development	30%
Phase 2: Navigation Implementation	20%
Phase 3: Firebase Backend	30%
Phase 4: Services/Sensors Integration	20%
Total	100%
Expected Learning Outcomes
Upon successful completion of this project, students will be able to:

Design responsive mobile user interfaces using Flutter.

Develop applications using Stateless and Stateful widgets.

Implement navigation between multiple application screens.

Connect Flutter applications with Firebase Authentication and Cloud Firestore.

Perform complete CRUD operations using Firestore.

Integrate mobile device services and hardware sensors.

Develop a complete cross-platform mobile application following modern software development practices.

This phased approach ensures that students gain practical experience in frontend development, navigation, backend integration, and device-level functionality while building a production-ready Flutter application.
